# QA Crypto Supplement

Дополнение к основному флоу (Jira → Figma → Allure). Используется когда нужно:
- составить регрессионный сьют по модулю
- сверить приоритизацию тест-кейсов
- проверить покрытие по крипто-специфике

> ⚠️ Для создания баг-репортов используй скилл **wallet-bugreport** — он создаёт задачи в Jira с нужными полями для Android / iOS / Web.

---

## Приоритизация тест-кейсов (P0–P3)

| Приоритет | Описание | Когда запускать |
|-----------|----------|-----------------|
| P0 | Критический путь: авторизация, отправка/получение крипты, отображение баланса, подключение биржи | Каждый билд |
| P1 | Основные фичи: история транзакций, конвертация пар, карточки активов, уведомления | Ежедневно / еженедельно |
| P2 | Вспомогательные функции: фильтры, сортировки, настройки профиля, UI-состояния | Еженедельно / релиз |
| P3 | Граничные и редкие кейсы: экстремальные суммы, нестандартные пары, edge cases UI | Только релиз |

---

## Крипто-специфичные проверки (чеклист)

Добавляй к тест-кейсам где применимо:

**Суммы и пары:**
- [ ] Очень маленькое значение (0.00000001 BTC / сатоши)
- [ ] Очень большое значение (999999.99999 BTC)
- [ ] Нулевое значение (0.00)
- [ ] Значение с максимальным количеством знаков после запятой
- [ ] Редкая / экзотическая торговая пара

**Задержки и состояния загрузки:**
- [ ] Отображение скелетона / лоадера пока данные подгружаются
- [ ] Поведение при медленном интернете (network throttling)
- [ ] Корректное обновление цены при волатильности (rapid price change)
- [ ] Состояние "нет данных" если биржа не вернула котировку

**Транзакции:**
- [ ] Pending-статус транзакции (ожидание подтверждения блокчейна)
- [ ] Failed-транзакция (insufficient funds, network error)
- [ ] Duplicate transaction prevention
- [ ] Отображение комиссии (gas fee / network fee)

**Безопасность и данные:**
- [ ] Маскировка приватных ключей / seed-фразы в UI
- [ ] Отображение адреса кошелька (полный vs сокращённый)
- [ ] Копирование адреса в буфер (clipboard)

## Шаблон регрессионного сьюта

Используй когда нужно покрыть целый модуль после изменений.

### Smoke (15–20 мин) — запускать при каждом деплое

```
SMOKE-001: Авторизация — пользователь входит в кошелёк
SMOKE-002: Главный экран — баланс отображается
SMOKE-003: Отправка — форма открывается, поля доступны
SMOKE-004: Получение — адрес кошелька отображается
SMOKE-005: История транзакций — список загружается
SMOKE-006: Биржа — переход на биржу работает
```

### P0 Targeted (30–45 мин) — после изменений в затронутом модуле

Пример для модуля **Send / Отправка крипты:**
```
TC-SEND-P0-001: Успешная отправка валидной суммы на корректный адрес
TC-SEND-P0-002: Блокировка отправки при недостаточном балансе
TC-SEND-P0-003: Отображение и подтверждение комиссии перед отправкой
TC-SEND-P0-004: Pending-статус после отправки до подтверждения
```

### Full Regression (2–3 ч) — перед релизом

Порядок выполнения:
1. Smoke → если падает, стоп
2. P0 → если падает, блокируем релиз
3. P1 → фиксируем падения, оцениваем риск
4. P2 / P3 → документируем, решаем по ситуации

## Критерии pass / fail для релиза

**✅ PASS:**
- Все P0 тест-кейсы зелёные
- ≥ 90% P1 тест-кейсов зелёные
- Нет открытых критических багов

**🚫 BLOCK (не выпускаем):**
- Любой P0 упал
- Баг с потерей данных или средств пользователя
- Уязвимость безопасности (приватные ключи, адреса)

**⚠️ CONDITIONAL (выпускаем с оговоркой):**
- P1 падения с задокументированным workaround
- Известная проблема внесена в release notes

## Анти-паттерны — чего не делать

| Избегай | Почему | Вместо |
|---------|--------|--------|
| "Баланс не отображается" | Нельзя воспроизвести | "При открытии вкладки Wallet на счёте с 0 транзакций баланс показывает '--' вместо '0.00 BTC'" |
| Тест-кейс без precondition | Тест падает по не той причине | Всегда указывай: авторизован / открыта страница / наличие баланса |
| Один тест на несколько проверок | Нельзя точно локализовать дефект | Один тест = одна проверка (ISTQB) |
| Игнорировать loading states | В крипте данные приходят с задержкой — это отдельный кейс | Покрывай skeleton / spinner отдельно |
| Округлять суммы в тестах | В крипте точность до 8 знаков критична | Используй реальные граничные значения |

## Быстрая сверка покрытия

Перед финальным прогоном проверь:

**Auth & Session:**
- [ ] Вход / выход
- [ ] Сессия после обновления страницы
- [ ] Поведение при истечении токена

**Wallet Core:**
- [ ] Отображение баланса (с данными / без / загрузка)
- [ ] Send flow (happy path + insufficient funds)
- [ ] Receive (адрес, QR)
- [ ] История транзакций (все статусы: pending, confirmed, failed)

**Exchange:**
- [ ] Переход на биржу
- [ ] Отображение пар и котировок
- [ ] Состояние при недоступности биржи

**UI / States:**
- [ ] Empty state (нет активов, нет истории)
- [ ] Error state (сеть недоступна)
- [ ] Loading state (skeleton/spinner)
- [ ] Соответствие Figma-макету
